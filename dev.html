Introduction

Welcome to the Dev Center for the Red Line Awards website. This document serves as the official developer documentation, detailing the technologies, implementation methods, and design considerations used to build the site.

Technology Stack

The Red Line Awards website leverages a modern tech stack to deliver a dynamic and interactive experience.

  • Vue.js 3: Used for building the reactive user interface and managing application state with components.
  • Supabase: Provides backend services including a PostgreSQL database, real-time subscriptions, and authentication.
  • GSAP: GreenSock Animation Platform for smooth animations and transitions.
  • Swiper.js: Implements sliders and carousels for main content and movie stills.
  • CSS3: Custom styling with fonts like Galmuri11 and Free License, and responsive layouts.

Component Structure

The site is built using Vue.js components, organized modularly for reusability and maintainability.

  • App: Root component managing layout and global state.
  • Navbar: Navigation bar with dropdown functionality.
  • Sidebar: Displays now-playing movies.
  • MainSlider: Main slider on the homepage using Swiper.js.
  • MovieGrid: Grid layout for movie posters.
  • MovieExpanded: Detailed view of a selected movie.
  • AwardSection: Displays award-winning movies.
  • Footer: Footer with links, including to this Dev Center.
  • ChatWindow: Live support chat window integrated with Supabase for real-time messaging.
  • SideBanners: Dynamic side banners that adjust position based on scroll to avoid footer overlap.

Database Integration

Supabase is used as the backend, managing data with real-time capabilities.

Tables

  • main_slider: Stores images for the main slider (image_url).
  • rla_recommendations: RLA recommended movies (id, title, poster_url, description).
  • rla_stills: Movie still images (movie_id, still_url).
  • movie_awards: Award-winning movies (id, movie_title, award_type, award_date, poster_url, short_description, still_cuts).
  • nominees: Nominee data (id, movie_title, award_category, rating).
  • notices: Notice board posts (id, title, content, created_at).
  • events: Event posts (id, title, content, created_at).
  • discussions: Movie discussion posts (id, title, content, created_at, updated_at). The 'updated_at' field is used to implement a floating post system where posts with recent comments float to the top.
  • discussion_comments: Comments on discussions (id, discussion_id, comment_text, unique_id, created_at, parent_id). Comments are ordered by 'created_at' and support nested replies using 'parent_id'.
  • soundtrack: Soundtrack posts (id, title, content, created_at, views).
  • masters_pick: Master's pick movies (id, title, content, image, views, created_at).
  • sidebanner: Side banner images (sidebanner_image).
  • chat_sessions: Chat session data (id, ip_address, last_user_read_at, last_admin_read_at, status).
  • chat_messages: Chat messages (id, session_id, message_text, is_admin, created_at).

Real-time subscriptions are implemented using Supabase’s PostgreSQL change events, e.g., for discussion comments: 

this.realtimeChannel = this.supabase
  .channel('discussion-changes')
  .on('postgres_changes', { event: 'INSERT', schema: 'public', table: 'discussion_comments' }, payload => {
    this.fetchDiscussions();
  })
  .on('postgres_changes', { event: 'UPDATE', schema: 'public', table: 'discussions' }, payload => {
    this.fetchDiscussions();
  })
  .subscribe();

The floating post system is achieved by sorting discussions based on the 'updated_at' field, which is updated whenever a new comment is added to a discussion.

Animations

GSAP powers the site’s animations, enhancing user interaction.

  • Intro Overlay: Fades out on site entry. 
    gsap.to("#intro-overlay", { duration: 1, opacity: 0, onComplete: () => { ... } });
  • Navbar Dropdown: Slides down/up on toggle. 
    gsap.to(nav, { duration: 0.3, height: nav.classList.contains("open") ? 44 : 340 });
  • Movie Expanded View: Scales in from the left. 
    gsap.fromTo("#movie-expanded-wrapper", { scaleX: 0 }, { duration: 0.5, scaleX: 1 });
  • Page Transitions: Scales the page container. 
    gsap.to("#page-container", { duration: 0.5, scale: 0, onComplete: () => { ... } });
  • Chat Window: Scales in/out on toggle. 
    gsap.to("#chat-window", { duration: 0.3, scale: this.showChatWindow ? 1 : 0 });

UI/UX Considerations

Red Line Awards ensures consistent usability and accessibility across its platform. However, the Dev Center is designed with a priority on usability and accessibility, taking inspiration from MDN Web Docs.

  • Responsive Design: Adapts to various screen sizes using media queries.
  • Clear Navigation: A navigation bar (Nav) and sidebar are provided for easy access to different sections..
  • Readable Text: Uses legible fonts and spacing.
  • Interactive Elements: Buttons and links styled for clarity.

Language Support

The Dev Center and Business pages support both English and Korean, toggled via a button.

Implementation uses data attributes and JavaScript to switch content: 

<p data-en="English text" data-kr="한국어 텍스트">English text</p>
<script>
  function toggleLanguage() {
    const lang = button.textContent === 'EN' ? 'kr' : 'en';
    document.querySelectorAll('[data-en][data-kr]').forEach(el => {
      const content = el.getAttribute(`data-${lang}`);
      el.innerHTML = content;
    });
  }
</script>

Chat System

A live support chat system is integrated using Supabase for real-time messaging.

Key features include session management and unread message indicators: 

async openChatWindow() {
  this.showChatWindow = true;
  let { data: sessions } = await this.supabase.from('chat_sessions').select('*').eq('ip_address', this.clientIP);
  if (sessions.length === 0) {
    const { data: newSession } = await this.supabase.from('chat_sessions').insert([{ ip_address: this.clientIP }]).select();
    this.chatSession = newSession[0];
  }
  this.setupRealtimeSubscription();
}

Side Banners

Dynamic side banners adjust position based on scroll to avoid footer overlap. 

function adjustBannerPosition() {
  const footerRect = footer.getBoundingClientRect();
  if (footerRect.top < windowHeight) {
    const overlap = windowHeight - footerRect.top;
    leftSidebanner.style.bottom = `${overlap + 260}px`;
  } else {
    leftSidebanner.style.bottom = 'max(20vh,10rem)';
  }
}
window.addEventListener('scroll', adjustBannerPosition);
